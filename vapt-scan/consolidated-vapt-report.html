Findings Detail (Warnings Only — No Critical/High Issues Found)

X-Content-Type-Options Header Missing Low Risk
Source: API Scan  |  CWE: 693  |  WASC: 15
Affected: /api/health, /openapi.json
The Anti-MIME-Sniffing header X-Content-Type-Options was not set to 'nosniff'. This allows older browsers to perform MIME-sniffing on the response body, potentially interpreting content as a different type than declared.
Recommendation: Add the X-Content-Type-Options: nosniff response header to all API endpoints. In FastAPI, add middleware: 
from starlette.middleware.base import BaseHTTPMiddleware

class SecurityHeadersMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        response = await call_next(request)
        response.headers["X-Content-Type-Options"] = "nosniff"
        return response

app.add_middleware(SecurityHeadersMiddleware)
Cross-Origin-Resource-Policy Header Missing or Invalid Low Risk
Source: API Scan  |  CWE: 693  |  WASC: 14
Affected: /api/health, /openapi.json
Cross-Origin-Resource-Policy header is an opt-in header designed to counter side-channel attacks like Spectre. Resources should be specifically set as shareable among different origins.
Recommendation: Add the Cross-Origin-Resource-Policy: same-origin header (or cross-origin if the API is accessed from different origins). Add to the same SecurityHeadersMiddleware: 
response.headers["Cross-Origin-Resource-Policy"] = "cross-origin"
Storable and Cacheable Content Informational
Source: Backend Baseline  |  CWE: 524  |  WASC: 13
Affected: Root path (404), /sitemap.xml (404)
404 error responses are cacheable by default. If error pages inadvertently contain sensitive info, they could be cached by proxies.
Recommendation: Add Cache-Control: no-cache, no-store, must-revalidate headers to error responses. This is a very low-priority informational finding.
Non-Storable Content Informational
Source: Frontend Baseline  |  CWE: 524  |  WASC: 13
Affected: Root path (403), /sitemap.xml (403)
Vite dev server returned 403 for root requests from ZAP's Docker container (expected in dev mode). Response content is not storable by caching components.
Recommendation: No action needed. This is expected behavior in development. In production, serve frontend via Nginx/CDN with proper cache headers.
